◆ CreateFile()

def fileHandling.views.CreateFile (   request)

Invoked when a POST request to "api/create-file/" is made.

Request should contain the keys 'data' (optional), 'filename' (necessary), 'parent' (optional, default = -1) and 'is_file' (optional, default = 'true') in request body. 'parent' is the file_id (a non negative integer) of folder in which the new file/folder needs to be created. For root folder as the parent, use -1.

Returns a JSON response with keys 'success' and 'message'. If file/folder is successfully created, then 'success' is set to True, and additional parameters namely 'file_id' and 'size' are returned for the newly created file/folder.

◆ ReturnDict()

def fileHandling.views.ReturnDict (   fid)

For folders, returns a dictionary with keys 'data' and 'children'.

'data' contains a dictionary with keys 'name', 'size', 'items', 'is_file', 'id' and 'path'. 'children' is a list of dictionaries obtained by calling this function recursively on the immediate children of the folder.

For files, returns a dictionary with a single key - 'data' which contains a dictionary with keys 'name', 'size', 'is_file', 'id' and 'path'.

@type fid = int

Parameters
fid= the file_id of file/folder for which the structure is required.

◆ UpdateFile()

def fileHandling.views.UpdateFile (   request)

Invoked when a POST request to "api/update-file/" is made.

Request should contain the keys 'file_id' (necessary), 'data' (optional) and 'filename' (optional) in request body. If 'filename' or 'data' is absent, they will not be updated, otherwise they will be overwritten.

Returns a JSON response with keys 'success' and 'message'. If file/folder is successfully updated, then 'success' is set to True, otherwise, it is set to False.
